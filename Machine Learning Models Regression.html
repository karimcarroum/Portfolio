This script predicts the win rate of professional league of legends’ players using in-game performance statistics.

Note that this script contains several models, which means that there is a lot of raw information, which can make the script long and repetitive. This is just for completeness of the analysis, so that everything can be replicated, properly justified and documented. A brief summary of the results is accompanies this script. For a faster read, less focused on the details but more on the results, the reader can jump to the last sections of the script (using the table of contents), where the performance of the models is evaluated. After that, insights are extracted using the most relevant variables for this regression and the predicitons on the test set.

3 Machine Learning Models

Cross validations to tune parameters are done over 10 folds using the train set, but if we want to reduce the computational time this can be reduced. We use the test set to evaluate the performance of each optimally tuned model trained in the train set, reporting mean absolute error and mean squared error of the predictions, and to obtain variable importance measures.

3.1 Random Forests and Bagging

We cross validate random forests to tune the number of variables used at each split (parameter “mtry”) of the model. Since the cross validation also evaluates the case where we consider all the variables at each split, this approach also assesses bagging (mtry = \(p\)). We could also tune by optimal number of trees and other parameters, but the computational time scales up quickly with each extra parameter, and the gains resulting from tuning them are not as big as the ones from tunning mtry.

require(randomForest); require(rfUtilities); require(tree); require(caret); require(pROC)

3.1.1 Parameter tunning

5.2 Create a categorical response

To cluster observations by levels of predictions done by the model, we create a categorical variable. Such clustering is complementary of the non-clustered analysis.

For simplicity, the intervals are set to be of equal range. Note that this can yield to having considerably different lengths of observations at each interval.

divide = function(x, nintervals = 5){ # Function to create equally divided intervals of the range of predictions. Will be used in the following function.
  range.x = max(x) - min(x); interval.range = range.x/nintervals; intervals.lower = rep(0,nintervals); intervals.lower[1] = min(x)
  for (interval in 2:nintervals){
    intervals.lower[interval] = intervals.lower[interval-1] + interval.range
  }
  return(intervals.lower)
}

cont.to.cat = function(x){ # Function to convert a continuous variable to a categorical with five categories, from very low to very high.
  intervals.mark = divide(x = x, nintervals = 5) # The function divide is the one created above.
  converted = rep(NA,nrow(test))
  converted[x <= intervals.mark[2]] = "Very Low"
  converted[x > intervals.mark[2] & x <= intervals.mark[3]] = "Low"
  converted[x > intervals.mark[3] & x <= intervals.mark[4]] = "Medium"
  converted[x > intervals.mark[4] & x <= intervals.mark[5]] = "High"
  converted[x > intervals.mark[5]] = "Very High"
  converted = as.factor(converted)
  return(list("converted"=converted, "intervals.mark"=intervals.mark[2:length(intervals.mark)]))
}
