This script predicts the main position of professional league of legends’ players using in-game performance statistics. The models predict with an accuracy of roughly 80%, with AUC scores of above 90%. Different machine learning methodologies are provided together with an aggregation rule for their results.

Since I evaluate a few models, there is a lot of raw information, which can make the script long and repetitive. This is just for completeness of the analysis, so that everything can be replicated, properly justified and documented. A brief summary of the results is accompanies this script. For a more efficient read, less focused on the details but more on the results, the reader can jump to the last sections of the script (using the table of contents), where all the predictions of the models are aggregated using a voting algorithm with a rule to break ties. After that, we extract insights using the most relevant variables for classification and the predicitons on the test set.

Note that the parameter tuning is simplified to improve computational times. In the section corresponding to each model, additional parameters that can be tunned are discussed.

Set timer for the computation of all models, diagnostics and exploratory insights using the results.

2 Dividing into Train and Test sets

The train set will be used to train the models via cross validated tunning of the parameters. The test set is also used to evaluate the relevance of the predictors in each model.

2.1 Create Train and Test sets

We define the sizes of the train and test sets to be the 50% of the whole dataset each.

seed= 1; train.size = round(nrow(dyf)*0.5)
set.seed(seed); train.ind = sample((1:nrow(dyf)),train.size); train = dyf[train.ind,]; test = dyf[-train.ind,]
sprintf("Train set size is %i and test set size is %i.", nrow(train), nrow(test))
## [1] "Train set size is 802 and test set size is 801."
target.train = train[,target.var]; target.test = test[,target.var] # Create a vector with the response for both sets, useful for the syntax of some functions.

2.2 Create a standardized version of the Train and Test sets

We scale the continuous predictors since it is required for a usually better performance of support vector machines and neural networks algorithms.

non.continous.predictors = c(); continuous.predictors = colnames(dfeatures)[colnames(dfeatures) %notin% non.continous.predictors]
dyf.scaled = data.frame(dyf[,target.var],dyf[,non.continous.predictors],scale(dyf[,continuous.predictors])); colnames(dyf.scaled)[1] = target.var

  colnames(dyf.scaled)[2:(2+length(non.continous.predictors)-1)] = non.continous.predictors

3 Machine Learning Models

Cross validations to tune parameters are done over 10 folds using the train set, but if we want to reduce the computational time this can be reduced. We use the test set to evaluate the performance of each model optimally tuned in the train set. We compute the associated confusion matrix and AUC score (area under the ROC curve). In the confusion matrix, we can evaluate the global accuracy of the model, as well as the accuracy for each class by checking the sensitivity scores. The higher the AUC score (area below the ROC curve), the better the trade-off between true and false positives.

3.1 Random Forests and Bagging

We cross validate random forests to tune the number of variables used at each split (parameter “mtry”) of the model. Since the cross validation also evaluates the case where we consider all the variables at each split, this approach also assesses bagging (mtry = \(p\)). We could also tune by optimal number of trees and other parameters, but the computational time scales up quickly with each extra parameter, and the gains resulting from tuning them are not as big as the ones from tunning mtry.

require(randomForest); require(rfUtilities); require(tree); require(caret); require(pROC)

3.1.1 Parameter tunning

3.2 Support Vector Machine

We cross validate to tune the cost and gamma parameters. The gamma parameter defines how far the influence of a single training example reaches, with low values meaning ‘far’ and high values meaning ‘close’.

require(e1071)

3.2.1 Parameter tunning

We use the standardized sets as it is convenient for the svm algorithm. For simplicity, we use a radial kernel.

set.seed(seed); svm.cv.start.time = Sys.time()
svm.cv = tune(svm, position~., data=train.scaled, kernel="radial", ranges=list(cost=c(0.1,1,10,100,1000),gamma=c(0.5,1,2,3,4)))
svm.cv.end.time = Sys.time(); svm.cv.total.time = svm.cv.end.time - svm.cv.start.time
sprintf("Support Vector Machine parameter tunning via cross validation takes %.2f seconds to compute.", round(svm.cv.total.time,2))
## [1] "Support Vector Machine parameter tunning via cross validation takes 36.46 seconds to compute."

Fitting the svm with the optimal cost and gamma parameters to the whole training set.

svm.fit=svm(position~., data=dyf.scaled, subset=train.ind, kernel="radial", cost=svm.cv$best.parameters[1], gamma=svm.cv$best.parameters[2], probability = TRUE)

We can choose the model with the best performance, or aggregate the predictions of all the models with an algorithm. I propose a majority rule, where in case of ties, the prediction used is that of the model with the best auc score. We could also use the accuracy scores instead.

class.decision.algorithm = function(class.predictions, algorithm.auc){
  decision = rep(NA,ncol(class.predictions)); best.auc.index = which.max(algorithm.auc)
  for (row in 1:nrow(test)){
    # First, define how to break majority rule ties.
    if(length(sort(table(class.predictions[,row])))>1){ # Only executes if there are different predictions for the same unit.
      if(sort(table(class.predictions[,row]),ascending=TRUE)[length(sort(table(class.predictions[,row])))]
       == sort(table(class.predictions[,row]),ascending=TRUE)[length(sort(table(class.predictions[,row])))-1]){ # Only executes if there is a majority rule tie.
        decision[row] = class.predictions[best.auc.index,row]
      }
      else{ # If there are different predictions for a unit, but there is no tie using the majority rule, use it.
        decision[row] = class.predictions[which.max(table(class.predictions[,row])),row]
      }  
    }
    # Decide using majority rule if there are no different predictions.
    else {
      decision[row] = class.predictions[which.max(table(class.predictions[,row])),row]
    } 
  }
  return(decision)
}
